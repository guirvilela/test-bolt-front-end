  <h3>Como iniciar o projeto</h3>
<ol>
  <li><strong>Instalar dependências:</strong> No terminal, execute o comando <code>npm install</code> para instalar todas as dependências do projeto.</li>
  <li><strong>Iniciar o servidor de desenvolvimento:</strong> Execute <code>npm run dev</code> para iniciar o servidor de desenvolvimento em <strong>http://localhost:3000</strong>.</li>
</ol>

<h3>Como executar os testes</h3>
<ol>
  <li><strong>Testes unitários:</strong> Para executar os testes unitários, use o comando <code>npm test</code>. Isso irá rodar os testes configurados com Jest.</li>
</ol>

<h3>Como executar o Storybook</h3>
<ol>
  <li><strong>Iniciar o Storybook:</strong> Para visualizar os componentes em desenvolvimento, execute <code>npm run storybook</code>. O Storybook será iniciado em <strong>http://localhost:6006</strong>.</li>
</ol>

<h3>Criar as tabelas no MySQL</h3>
<p>Para configurar o banco de dados MySQL, execute os seguintes comandos para criar as tabelas necessárias:</p>
<pre>
  CREATE TABLE `operations` (
      `id` INT AUTO_INCREMENT PRIMARY KEY,
      `userId` INT,
      `type` VARCHAR(255) NOT NULL,
      `amount` DECIMAL(10,2) NOT NULL,
      `senderId` INT,
      `timestamp` DATETIME DEFAULT CURRENT_TIMESTAMP,
      `balanceAfter` DECIMAL(10,2),
      `recipientId` INT,
      `revertId` INT,
      `recipientName` VARCHAR(255)
  );

  CREATE TABLE `users` (
      `id` INT AUTO_INCREMENT PRIMARY KEY,
      `username` VARCHAR(255) NOT NULL,
      `password` VARCHAR(255) NOT NULL,
      `email` VARCHAR(255) NOT NULL UNIQUE,
      `balance` DECIMAL(10,2) DEFAULT 0.00
  );
</pre>

<h3>Configuração do .env</h3>
<p>Não se esqueça de criar um arquivo <strong>.env</strong> na raiz do projeto com as seguintes variáveis de ambiente para configurar o MySQL:</p>
<pre>
  MYSQL_ROOT_PASSWORD=rootpassword
  MYSQL_DATABASE=authdb
  MYSQL_USER=user
  MYSQL_PASSWORD=userpassword
</pre>

<h3>Como usar o Docker para integrar o front-end e back-end</h3>
<ol>
  <li><strong>Iniciar o container do Docker:</strong> Use o <code>docker-compose up</code> para iniciar o front-end e o back-end no Docker. Certifique-se de ter um <code>docker-compose.yml</code> configurado adequadamente com a integração entre os serviços.</li>
  <li><strong>Verificar os containers:</strong> Após a execução, você pode verificar os containers com <code>docker ps</code> para garantir que tudo está funcionando corretamente.</li>
</ol>
